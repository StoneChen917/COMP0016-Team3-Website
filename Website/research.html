Research

In our research, we considered many concepts including what they are, their different types and why they are the best suited for our project.


Named Entity Recognition

What is Named Entity Recognition?
Named Entity Recognition is mainly the process of detecting entities from text and classifying them into different categories that we can use.

Different types of NER:
1. Dictionary-based has a dictionary containing vocabulary which is used. Basic string-matching algorithms check whether the entity is present in the given text against the items in the vocabulary. This method is generally not employed because the dictionary that is used is required to be updated and maintained consistently.
2. Rule-based has a predefined set of rules for information extraction is used which are pattern-based and context-based. Pattern-based rule uses the morphological pattern of the words while context-based uses the context of the word given in the text document.
3. Machine learning based solves a lot of limitations of the above two methods. It is a statistical-based model that tries to make a feature-based representation of the observed data. It can recognise an existing entity name even with small spelling variations. The machine learning-based approach involves two phases for doing NER. The first phase trains the ML model on annotated documents. In the next phase, the trained model is used to annotate the raw documents. The process is similar to a normal ML model pipeline.
4. Deep learning NER is more accurate than the ML-based method because it is capable of assembling words, enabling it to understand the semantic and syntactic relationship between various words better. It is also capable of analysing topic-specific and high-level words automatically.

Why we implemented NER:
Our client has asked us to prioritise the extraction of information related to Admin 1 and 2 as well as geographical information such as ISO codes so that they can update an already existing database with additional extracted information, therefore we believe the most appropriate approach to go with is NER.


SpaCy

What is SpaCy?
SpaCy is an open-sourced Natural Language Processing library for Python. spaCy is designed to make it easy to build systems for information extraction or general-purpose natural language processing. SpaCy utilises tokens which breaks a text down into its basic units which are then represented in spaCy as Token objects. Tokenization is a process of splitting up text into smaller parts(tokens). Tokenisation can be either words, characters or subwords:
Word tokenization splits text into individual words based on the space.
Character tokenization splits text into sets of characters. These characters will represent words.
Subword tokenization splits text into subwords. These subwords can be combined to create whole words.


Q&A Model

What is QA model?
Question-Answering models are machine learning tools that can provide answers to given questions once provided some information/ context. It is a form of Natural Language Processing where the computer attempts to “understand” natural language and interpret them.

Different types of QA Model:
1.Reading Comprehension aims to provide answers to questions through understanding of words and concepts. This is best suited for literary (eg. paper and articles).
2.Closed book questioning involves a computer learning a set of facts and answering questions based on those facts.
3.Closed-domain uses questions under a specific domain which limits the questions that can be asked to those in a specific category.
4.Open-domain uses a much larger dataset to answer questions, and therefore has a larger scope of questions that can be asked.

Hugging Face:
Hugging Face is a platform that allows users to shared pre-trained models that can be used for Natural Language Processing. It is popular for its utilisation of the transformer library in Python and its use of models such as BERT and RoBERTa.

Why we implemented QA model:
We implemented a QA model because it accurately was able to use keyword matching and information retrieval to answer many of the questions we needed for for our requirements (eg. operation start date, operation end date, glide number, operation budget).


Table Extraction

What is Table Extraction?
Table extraction is a concept that, when implemented, can extract specific data from a table. Certain libraries allow you to convert unstructured data in PDF files to clean, structured data.

What is Camelot?
Camelot is a Python library that allows you to extract tables from PDFs.
Why Camelot?
Configurability: Camelot gives you control over the table extraction process with tweakable settings.
Metrics: You can discard bad tables based on metrics like accuracy and whitespace, without having to manually look at each table.
Output: Each table is extracted into a pandas DataFrame, which seamlessly integrates into ETL and data analysis workflows. You can also export tables to multiple formats, which include CSV, JSON, Excel, HTML, Markdown, and Sqlite.

What is PyPDF2:
PyPDF2 is a free and open-source pure-python PDF library capable of splitting, merging, cropping, and transforming the pages of PDF files.


String Matching

What is String Matching?
String matching refers to finding occurrence(s) of a pattern string within another string or body of text.

Types of String Matching:
1.Exact String Matching is to find one, several, or all occurrences of a defined string (pattern) in a large string (text or sequences) such that each matching is perfect.
2.Approximate String Matching Algorithms (also known as Fuzzy String Searching) searches for substrings of the input string.

Fuzzy matching:
Fuzzy matching is a method that provides an improved ability to process word-based matching queries to find matching phrases or sentences from a database. When an exact match is not found for a sentence or phrase, fuzzy matching can be applied. This improves accuracy as it can identify false-positives and remove redundancies.

Cosine Matching:
Cosine Matching addresses the similarity between two things and assigns it a numeric value of similarity (1 being the most similar and 0 being the least). It can be used when fuzzy matching becomes too probabilistic. In our project this can help when finding the admin codes as cosine similarity can be used on admin 0 codes to reduce the lengthy size of the ISO codes list to make it easier to find admin 1 and 2.

String Matching Comparisons:
We decided to compare the 3 types of string matching on the basis of time taken to execute the program, the number of ‘Not Found’ values returned as well as the accuracy which was calculated by (number of correct answers / total number of answers) * 100.
It was observed that adding the first letter filter did not reduce time of execution significantly except in the case of Cosine Matching. However, the additional filter reduced accuracy in all cases except with Cosine Matching as well as increased the number of ‘Not Found’ values returned from 0 to 1 in all cases. This is because locations have been entered with different names in the spreadsheets and these entries were filtered out and therefore not considered when string matching was performed(For e.g.: Johannesburg has been entered as City of Johannesburg).

Tkinter

What is Tkinter
Tkinter is the Tk GUI toolkit for Python. It provides graphical widgets such as menus, buttons, data fields for building application interfaces.

Why we chose Tkinter
Tkinter is widely considered to be the best GUI (Graphical User Interface) library for Python due to several key reasons. Firstly, it is an in-built library that comes with Python, meaning there is no need to install or configure any external libraries to use it. This makes it extremely convenient and accessible for developers, particularly those who are new to Python programming.
-Tkinter is very user-friendly and easy to learn. The library provides a simple and intuitive way to create graphical interfaces for desktop applications, making it an ideal choice for beginners. The widgets and tools provided by Tkinter are designed to be easy to use and understand, with a clear and consistent structure that makes it easy to create custom GUI designs.
-Another advantage of Tkinter is its cross-platform compatibility. It works seamlessly on Windows, Linux, and MacOS, making it an ideal choice for developers who need to create applications that can run on different operating systems. This versatility ensures that the applications developed using Tkinter have a broad reach, and can be used by a large number of users across different platforms.
-Tkinter is an actively developed and maintained library with a vast online community. This means that developers can easily find support, documentation, and tutorials on the library. Moreover, there are several third-party libraries and tools available that can be used in conjunction with Tkinter to enhance its functionality and capabilities.
In conclusion, the convenience, ease of use, cross-platform compatibility, and strong community support make Tkinter the best choice for Python GUI development.

PostgreSQL

What is PostgreSQL
PostgreSQL, also known as Postgres. It is an open-source relational database management system

Why we chose PostgreSQL
PostgreSQL is a powerful and feature-rich open-source relational database management system (RDBMS) that has gained popularity among developers and businesses worldwide. It has several features that make it the best choice for a project.
-PostgreSQL is highly reliable and stable. It has a proven track record of providing high performance and availability, even for complex and large-scale applications. Its robust transactional model ensures that data is consistently stored and protected, making it an excellent choice for mission-critical applications.
-PostgreSQL is highly extensible and customizable. It provides a wide range of data types, functions, and extensions that can be used to support various data structures and processing requirements. Additionally, its flexible indexing system enables developers to create indexes that are tailored to specific query patterns, thereby improving query performance.
-PostgreSQL is highly secure. It provides several authentication and authorization mechanisms, including SSL encryption, LDAP, and Kerberos, to ensure that data is protected from unauthorized access and attacks.
-PostgreSQL is highly scalable. It can efficiently handle large volumes of data and support a significant number of concurrent users. This makes it an ideal choice for projects that require scalability and high availability.
-PostgreSQL has a large and active community of developers and users. It has a wide range of documentation, tutorials, and user groups that make it easy to learn and get help with any issues or problems that may arise during development.
In conclusion, PostgreSQL is the best choice for a project due to its reliability, extensibility, security, scalability, and strong community support.
